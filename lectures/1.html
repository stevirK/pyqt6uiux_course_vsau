Краткая история GUI

Графический интерфейс пользователя имеет долгую историю, уходящую корнями в 1960-е годы. Система NLS Стэнфордского университета впервые публично представила концепции мыши и окон в 1968 году. За ней последовал графический интерфейс Smalltalk из Xerox PARC (1973), ставший основой для большинства современных GUI.

В ранних системах уже присутствовало многое из того, что мы воспринимаем как должное сегодня — окна, меню, радиокнопки, флажки и значки. Эти элементы породили аббревиатуру WIMP (Windows, Icons, Menus, Pointing device).

В 1979 году вышла первая коммерческая система с GUI — рабочая станция PERQ, что дало толчок разработкам Apple Lisa (1983), Windows 1.0 (1985), X Window для UNIX (1984) и другим. Несмотря на постоянные попытки «революции» — от Windows 95 до macOS X и Windows 10 — парадигма WIMP оставалась стандартом 40 лет и продолжает доминировать на десктопе сегодня.

Немного о Qt

Qt — бесплатный инструментарий с открытым исходным кодом для создания кроссплатформенных приложений с GUI. Он позволяет разрабатывать приложения для Windows, macOS, Linux и Android из единой кодовой базы. Qt — это больше чем набор виджетов: он включает встроенную поддержку мультимедиа, баз данных, векторной графики и MVC-интерфейсов.

Qt основан Эйриком Чамбе-Энгом и Хавардом Нордом в 1991 году, первая компания Trolltech — в 1994-м. Сегодня разработку ведёт The Qt Company.

Qt и PySide6

PySide6 (он же Qt for Python) — официальная привязка Python к Qt, разрабатываемая The Qt Company. Когда вы пишете на PySide6, вы по сути создаёте Qt-приложение: PySide6 является обёрткой вокруг C++-библиотеки Qt.

Замечание о стиле кода: PySide6 использует соглашения об именовании Qt (mixedCase), а не Python-стандарт PEP8 (snake_case). Рекомендуется придерживаться PEP8 в собственном коде, чтобы чётко отличать его от кода библиотеки.

Создание первого приложения

Создайте новый файл Python (например, myapp.py) и введите следующий код:

myapp.py 
from PySide6.QtWidgets import QApplication, QWidget
import sys

# Нужен один (и только один) экземпляр QApplication на приложение.
# Передайте sys.argv для поддержки аргументов командной строки.
app = QApplication(sys.argv)

# Создаём виджет Qt — он станет нашим окном.
window = QWidget()
window.show()  # ВАЖНО: окна по умолчанию скрыты!

# Запускаем цикл обработки событий.
app.exec()

Запустить приложение можно из командной строки:

python myapp.py
# или для Python 3:
python3 myapp.py

Пошаговый разбор кода

Сначала импортируем нужные классы: QApplication — обработчик приложения, QWidget — базовый пустой виджет GUI из модуля QtWidgets.

Основные модули Qt: QtWidgets, QtGui и QtCore.

Создаём QApplication, передавая sys.argv (аргументы командной строки). Если аргументы не нужны, можно передать пустой список: QApplication([]).

Затем создаём экземпляр QWidget и вызываем window.show(). В Qt все виджеты верхнего уровня без родителя являются окнами — технически окном может быть любой виджет.

Цикл обработки событий

Ядром каждого Qt-приложения является QApplication. Именно он содержит цикл обработки событий — главный цикл, управляющий всем взаимодействием с GUI.

Каждое взаимодействие (нажатие клавиши, щелчок мыши, движение курсора) порождает событие, помещаемое в очередь. На каждой итерации цикл проверяет очередь, передаёт событие нужному обработчику, тот выполняет работу и возвращает управление циклу. В приложении всегда один активный цикл событий.

QMainWindow

В Qt любой виджет может стать окном. Например, QPushButton в роли окна: 

import sys
from PySide6.QtWidgets import QApplication, QPushButton

app = QApplication(sys.argv)
window = QPushButton("Push Me")
window.show()
app.exec()

Это интересно, но не очень практично. Для реальных приложений Qt предлагает QMainWindow — готовый виджет со стандартными функциями: панели инструментов, меню, строка состояния, закрепляемые панели и многое другое.

Лучший подход — создать подкласс QMainWindow и настраивать окно в методе __init__:

mainwindow.py 
import sys
from PySide6.QtCore import QSize, Qt
from PySide6.QtWidgets import QApplication, QMainWindow, QPushButton

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("My App")

        button = QPushButton("Press Me!")
        # Устанавливаем виджет в центральную область окна
        self.setCentralWidget(button)

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()

Изменение размеров окон и виджетов

По умолчанию окно можно свободно менять в размерах. Для ограничений Qt использует объект QSize(ширина, высота): 

import sys
from PySide6.QtCore import QSize
from PySide6.QtWidgets import QApplication, QMainWindow, QPushButton

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("My App")
        button = QPushButton("Press Me!")
        self.setFixedSize(QSize(400, 300))  # фиксированный размер
        self.setCentralWidget(button)

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()

Помимо .setFixedSize() доступны:

  • .setMinimumSize(QSize(w, h)) — минимальный размер окна
  • .setMaximumSize(QSize(w, h)) — максимальный размер окна
