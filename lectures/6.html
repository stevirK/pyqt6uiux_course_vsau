Лекция 6

Несколько окон:
создание, отображение и связь

🪟 Темы: QWidget без родителя, постоянные окна, show/hide, сигналы между окнами ⏱ ~35 мин

Отличие окон от диалогов

В предыдущей лекции мы рассмотрели модальные диалоги (QDialog), которые блокируют родительское окно до своего закрытия. Однако часто требуется открыть новое окно, которое не блокирует главное приложение — например, для отображения графиков, логов или работы с несколькими документами.

В Qt любое окно — это виджет без родителя (parent = None). Если у виджета нет родителя, он становится самостоятельным окном верхнего уровня.

Ключевое правило: Виджет без родителя → окно верхнего уровня.
Виджет с родителем → вложенный элемент внутри родительского окна.

Создание нового окна

Создадим простое приложение, которое открывает второе окно по нажатию кнопки.

simple_second_window.py 
import sys
from PySide6.QtWidgets import QApplication, QMainWindow, QPushButton, QLabel, QVBoxLayout, QWidget

class AnotherWindow(QWidget):
    """Окно без родителя — будет плавающим."""
    def __init__(self):
        super().__init__()
        layout = QVBoxLayout()
        self.label = QLabel("Другое окно")
        layout.addWidget(self.label)
        self.setLayout(layout)

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Главное окно")
        
        button = QPushButton("Открыть новое окно")
        button.clicked.connect(self.show_new_window)
        self.setCentralWidget(button)
    
    def show_new_window(self):
        w = AnotherWindow()
        w.show()

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()
⚠️ Проблема: Окно появляется на долю секунды и сразу исчезает!
Переменная w — локальная. Как только метод show_new_window завершается, сборщик мусора Python уничтожает объект окна, и оно закрывается.

Сохранение ссылки на окно

Чтобы окно не уничтожалось, нужно сохранить ссылку на него, например, как атрибут главного окна: 

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.w = None          # Ссылка на дополнительное окно
        # ... остальной код ...
    
    def show_new_window(self):
        if self.w is None:
            self.w = AnotherWindow()
        self.w.show()

Теперь окно создаётся один раз и остаётся в памяти. При повторном нажатии кнопки .show() просто делает его видимым (если оно было скрыто).

Важно: Если вы хотите полностью удалять окно при закрытии и создавать заново, используйте атрибут self.w = None и проверяйте его перед созданием.

Управление видимостью: show() и hide()

Для постоянных окон удобно переключать видимость, не создавая их каждый раз заново.

toggle_window.py 
import sys
from PySide6.QtWidgets import QApplication, QMainWindow, QPushButton, QLabel, QVBoxLayout, QWidget

class AnotherWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Дополнительное окно")
        layout = QVBoxLayout()
        self.label = QLabel("Это дополнительное окно")
        layout.addWidget(self.label)
        self.setLayout(layout)

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Главное окно")
        
        # Создаём дополнительное окно один раз
        self.extra_window = AnotherWindow()
        
        button = QPushButton("Показать/Скрыть окно")
        button.clicked.connect(self.toggle_window)
        self.setCentralWidget(button)
    
    def toggle_window(self):
        if self.extra_window.isVisible():
            self.extra_window.hide()
        else:
            self.extra_window.show()

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()
🎨 Теперь при каждом нажатии кнопки дополнительное окно показывается или скрывается. Окно сохраняет своё состояние (включая содержимое) при скрытии.

Закрытие окна через установку None

Альтернативный подход — полностью удалять окно при закрытии, устанавливая ссылку в None: 

def show_new_window(self):
    if self.w is None:
        self.w = AnotherWindow()
        self.w.show()
    else:
        self.w.close()
        self.w = None   # Окно будет уничтожено сборщиком мусора

Сигналы между окнами

Одно из главных преимуществ сохранения ссылки на окно — возможность напрямую подключать сигналы одного окна к слотам другого.

inter_window_signals.py 
import sys
from PySide6.QtWidgets import (
    QApplication, QMainWindow, QPushButton, QLabel, 
    QVBoxLayout, QWidget, QLineEdit
)

class AnotherWindow(QWidget):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Дополнительное окно")
        layout = QVBoxLayout()
        self.label = QLabel("Здесь будет текст")
        layout.addWidget(self.label)
        self.setLayout(layout)

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Главное окно")
        
        # Создаём дополнительное окно
        self.extra_window = AnotherWindow()
        
        # Элементы управления
        self.button = QPushButton("Показать/Скрыть окно")
        self.button.clicked.connect(self.toggle_window)
        
        self.input_field = QLineEdit()
        self.input_field.setPlaceholderText("Введите текст...")
        
        # ⭐ КЛЮЧЕВОЙ МОМЕНТ: сигнал textChanged из главного окна
        # подключается к слоту setText в дополнительном окне
        self.input_field.textChanged.connect(self.extra_window.label.setText)
        
        # Компоновка
        layout = QVBoxLayout()
        layout.addWidget(self.button)
        layout.addWidget(self.input_field)
        container = QWidget()
        container.setLayout(layout)
        self.setCentralWidget(container)
    
    def toggle_window(self):
        if self.extra_window.isVisible():
            self.extra_window.hide()
        else:
            self.extra_window.show()

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()
Результат: Текст, вводимый в поле главного окна, мгновенно отображается в метке дополнительного окна — даже если дополнительное окно скрыто! Обновление виджетов не зависит от их видимости.

Пример: приложение с логом событий

Более сложный пример: главное окно отправляет сообщения в отдельное окно-лог, которое можно показывать и скрывать.

log_window_example.py 
import sys
from PySide6.QtWidgets import (
    QApplication, QMainWindow, QPushButton, QLabel, 
    QVBoxLayout, QWidget, QTextEdit, QHBoxLayout
)
from PySide6.QtCore import Qt
from datetime import datetime

class LogWindow(QWidget):
    """Окно для отображения лога сообщений."""
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Лог событий")
        self.resize(400, 300)
        
        layout = QVBoxLayout()
        self.log_text = QTextEdit()
        self.log_text.setReadOnly(True)
        layout.addWidget(self.log_text)
        self.setLayout(layout)
    
    def add_message(self, message):
        timestamp = datetime.now().strftime("%H:%M:%S")
        self.log_text.append(f"[{timestamp}] {message}")

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.setWindowTitle("Главное приложение")
        
        # Создаём окно лога
        self.log_window = LogWindow()
        
        # Кнопка для показа/скрытия лога
        self.toggle_log_btn = QPushButton("Показать лог")
        self.toggle_log_btn.clicked.connect(self.toggle_log)
        
        # Кнопки для генерации событий
        self.info_btn = QPushButton("Info")
        self.info_btn.clicked.connect(lambda: self.log_window.add_message("Нажата кнопка Info"))
        
        self.warning_btn = QPushButton("Warning")
        self.warning_btn.clicked.connect(lambda: self.log_window.add_message("ВНИМАНИЕ: нажата Warning!"))
        
        self.error_btn = QPushButton("Error")
        self.error_btn.clicked.connect(lambda: self.log_window.add_message("ОШИБКА: что-то пошло не так"))
        
        # Компоновка
        layout = QVBoxLayout()
        layout.addWidget(self.toggle_log_btn)
        layout.addWidget(self.info_btn)
        layout.addWidget(self.warning_btn)
        layout.addWidget(self.error_btn)
        
        container = QWidget()
        container.setLayout(layout)
        self.setCentralWidget(container)
        
        # Логируем запуск
        self.log_window.add_message("Приложение запущено")
    
    def toggle_log(self):
        if self.log_window.isVisible():
            self.log_window.hide()
            self.toggle_log_btn.setText("Показать лог")
        else:
            self.log_window.show()
            self.toggle_log_btn.setText("Скрыть лог")

app = QApplication(sys.argv)
window = MainWindow()
window.show()
app.exec()
🎨 Результат: Окно лога можно показывать и скрывать. Все действия в главном окне добавляют сообщения в лог. Лог продолжает накапливать сообщения даже в скрытом состоянии.

Создание нескольких независимых окон

Вы можете создавать множество окон, например, для каждого открытого документа. 

class MainWindow(QMainWindow):
    def __init__(self):
        super().__init__()
        self.windows = []  # Список всех открытых окон
        
        button = QPushButton("Новое окно")
        button.clicked.connect(self.create_new_window)
        self.setCentralWidget(button)
    
    def create_new_window(self):
        new_win = AnotherWindow()
        new_win.setWindowTitle(f"Окно #{len(self.windows) + 1}")
        new_win.show()
        self.windows.append(new_win)
Совет по архитектуре: Для больших проектов лучше создавать отдельный класс-контроллер, который управляет окнами и связями между ними, а не хранить всё в главном окне.
Итог лекции:
  • Окно в Qt — это любой виджет без родителя
  • Для сохранения окна нужно хранить ссылку на него (атрибут или список)
  • .show() — показывает окно, .hide() — скрывает
  • .isVisible() проверяет видимость
  • Сигналы между окнами работают напрямую — достаточно иметь ссылку на объект
